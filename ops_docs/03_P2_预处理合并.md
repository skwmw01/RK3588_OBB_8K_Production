# P2: 预处理流水线合并（待实施）

## 优化类型

**内存访问优化** — 减少图像预处理阶段的内存分配和拷贝次数。

## 涉及算子

| 算子 | 当前实现 | 耗时占比 |
|------|---------|---------|
| Crop | `img[y:y+TILE_CUT, x:x+TILE_CUT]` NumPy 切片 | ~1% |
| ResizeBilinear | `cv2.resize(crop, (640, 640))` | ~3% |
| ColorConvert_BGR2RGB | `cv2.cvtColor(small, cv2.COLOR_BGR2RGB)` | ~1% |

## 瓶颈分析

当前流水线有 **3 次独立的内存分配**：

```python
# production_v2.py 第 55-58 行
crop = img[y:y+TILE_CUT, x:x+TILE_CUT]      # 拷贝 1：2048×2048×3 = 12MB
small = cv2.resize(crop, (TILE_MODEL, TILE_MODEL))  # 拷贝 2：640×640×3 = 1.2MB
rgb = cv2.cvtColor(small, cv2.COLOR_BGR2RGB)  # 拷贝 3：640×640×3 = 1.2MB
```

每次操作都要：
1. 分配新内存（malloc）
2. 读取源数据
3. 写入目标数据
4. 释放源数据（上一步的临时结果）

## 优化方案

### 方案 A：减少内存分配次数

```python
# 预分配输出缓冲区（复用）
_buf = np.empty((TILE_MODEL, TILE_MODEL, 3), dtype=np.uint8)

# 直接 resize 到预分配缓冲区
cv2.resize(img[y:y+TILE_CUT, x:x+TILE_CUT], (TILE_MODEL, TILE_MODEL), dst=_buf)
# 原地颜色转换（省一次分配）
cv2.cvtColor(_buf, cv2.COLOR_BGR2RGB, dst=_buf)
rgb = _buf
```

### 方案 B：UMat (OpenCL GPU 加速)

```python
# 利用 Mali-G610 GPU 做 resize + cvtColor
umat = cv2.UMat(img[y:y+TILE_CUT, x:x+TILE_CUT])
small = cv2.resize(umat, (TILE_MODEL, TILE_MODEL))
rgb = cv2.cvtColor(small, cv2.COLOR_BGR2RGB)
rgb = rgb.get()  # 从 GPU 取回 CPU
```

### 方案 C：NV12 直通（需要相机配合）

```python
# 如果相机输出 NV12，跳过 BGR→RGB 转换
# RKNN 支持 NV12 输入格式
inp = nv12_crop_resize(img[y:y+TILE_CUT, x:x+TILE_CUT])
outs = r.inference(inputs=[inp], data_format="nv12")  # 直接 NV12 输入
```

## 预期收益

| 方案 | 预期加速 | 改动量 | 风险 |
|------|---------|-------|------|
| A: 缓冲区复用 | ~1.2x | 小 | 低 |
| B: UMat GPU | ~1.5x | 中 | 需验证 Mali 兼容性 |
| C: NV12 直通 | ~2.0x | 大 | 需相机支持 NV12 |

**预估**：预处理从 ~25ms 降到 ~13-20ms，总耗时节省 ~5-12ms。
