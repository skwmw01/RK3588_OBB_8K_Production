## 优化类型

**查表优化 (Lookup Table)** — 将浮点数学运算替换为数组索引，用空间换时间。

## 涉及算子

| 算子 | 文件 | 当前实现 | 特征 |
|------|------|---------|------|
| Sigmoid | `code/common.py:sigmoid` | `1.0 / (1.0 + np.exp(-x))` | 逐元素 exp 运算 |
| Softmax | `code/common.py:decode_generic` 内 | `np.exp(xywh - max) / sum` | exp + sum |
| Exp | 同上 | `np.exp(...)` | 被 softmax 内联使用 |

## 瓶颈分析

### Sigmoid

```python
def sigmoid(x):
    return 1.0 / (1.0 + np.exp(-x))
```

- `np.exp` 对每个元素做浮点指数运算
- 在 decode_generic 中被调用 3 次（每个 stride 一次）
- 输入是 INT8 量化后的分类 logits，值域有限

### Softmax (DFL 部分)

```python
sm = np.exp(xywh - xywh.max(axis=1, keepdims=True))
sm /= sm.sum(axis=1, keepdims=True)
```

- 4×16×H×W 的 exp 运算
- 已做数值稳定性优化（减去 max）
- 16 个 bin 的 softmax，输入值域相对固定

## 优化方案

### Sigmoid LUT（INT8 输入专用）

```python
# 预计算 256 项 LUT（INT8 值域 -128~127 映射到 sigmoid 输出）
# 需要知道量化 scale/zero_point
def build_sigmoid_lut(scale=1.0/32, zero_point=-128):
    x = np.arange(256, dtype=np.float32)
    x_real = (x + zero_point) * scale
    return (1.0 / (1.0 + np.exp(-x_real))).astype(np.float32)

sigmoid_lut = build_sigmoid_lut()

# 调用：O(1) 查表替代 O(N) exp 运算
def sigmoid_fast(x_int8):
    return sigmoid_lut[x_int8.astype(np.uint8)]
```

**加速原理**：
- 原版：N 次 `exp()` 浮点运算
- LUT：N 次数组索引（整数加法 + 内存读取）
- 数组索引比 exp 快 10-50 倍

### Softmax LUT（16-bin 固定）

```python
# DFL 的 16-bin softmax，输入 shape 固定为 (4, 16, H, W)
# 16 个 bin 的 softmax 可以预计算所有可能的输入组合
# 但输入是连续浮点值，无法穷举 → 用近似多项式替代

# 方案：用多项式近似 exp（比 np.exp 快 3-5x）
def fast_exp(x):
    """多项式近似 exp，精度误差 < 1%"""
    x = np.clip(x, -10, 10)
    x2 = x * x
    return 1.0 + x * (1.0 + x * (0.5 + x * (1.0/6 + x * (1.0/24))))
```

## 预期收益

| 算子 | 原耗时 | LUT 后 | 加速比 |
|------|--------|-------|--------|
| Sigmoid (3次调用) | ~3ms | ~0.3ms | **10x** |
| Softmax (DFL) | ~5ms | ~2ms | **2.5x** |
| **总计** | ~8ms | ~2.3ms | **3.5x** |

> 注意：softmax 和 sigmoid 在 decode_generic 总耗时中占比不大（~8ms vs 总 285ms），因为主要瓶颈在 Python 循环（已由 P0+P1 优化）。

## 适用场景

LUT 最适合：
1. **输入值域有限**（如 INT8 的 256 个值）
2. **函数计算昂贵**（如 exp、tanh、erf）
3. **调用频率高**（如每像素每通道）

本项目中 sigmoid 输入是量化后的 INT8 值，完美符合 LUT 条件。

## 风险

- **精度损失**：LUT 有量化误差（取决于表大小和插值方式）
- **额外内存**：256 项 float32 LUT = 1KB，可忽略
- **需要量化参数**：LUT 构建依赖模型的 scale/zero_point，不同模型需重新生成
