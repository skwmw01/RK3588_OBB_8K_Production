## 🔍 各阶段性能详解

### Stage 1 · Tile 规划（0.05 ms）

```python
def gen_tiles(W=8192, H=8192, tile=2048, overlap=200):
    stride = 1848
    xs = [0, 1848, 3696, 5544, 5952]   # 最后一片靠右边界
    ys = [0, 1848, 3696, 5544, 5952]   # 最后一片靠下边界
    return [(x, y) for y in ys for x in xs]   # 25 个
```

忽略不计。

### Stage 2 · 预处理（~40 ms）

每 tile 3 件事：

1. crop（内存视图）：0.1 ms
2. resize 2048→1024 LINEAR：1.5 ms
3. BGR→RGB cvtColor：2 ms

单 tile ~3.5 ms × 25 = ~87 ms 串行，3 核压到 ~40 ms wall。

RK3588 CPU（4×A76 + 4×A55）配合 OpenCV NEON SIMD 加速。

### Stage 3 · NPU 推理（~440 ms）

单核推理 @1024 INT8：

| 项 | 耗时 |
|---|---|
| 输入 quantize | ~2 ms |
| 卷积前向（NPU） | ~45 ms |
| 输出 dequantize + 传 CPU | ~10 ms |
| **单推理** | **~55 ms** |

3 核并发理论下限：25/3 × 55 = **458 ms**
实测 ~440 ms（扩展率 111%，超线性）。

**为什么超线性？** 3 核跑不同 tile 数据互不争 L2 cache，反而更好。

### Stage 4 · OBB Decode（~285 ms）

**最大的非必要开销**。

对每个 tile 4 个输出：

```python
# outs[0]: (1, 79, 128, 128)  stride=8
# outs[1]: (1, 79, 64, 64)    stride=16
# outs[2]: (1, 79, 32, 32)    stride=32
# outs[3]: (1, 1, 21504)      angle

# 每个 stride：
# 1. 64 通道 reshape (4, 16, H, W) DFL bins
# 2. 16 bins softmax → 期望值 → (l,t,r,b) 偏移
# 3. 15 cls logits sigmoid → argmax + max conf
# 4. 筛 conf > 0.15
# 5. angle 查表 + 坐标映射
```

Python + numpy，单 tile ~32 ms × 8-9 = 250-290 ms。

**为什么慢**：
- Stride 8 级 H×W=128×128=16384 anchor 全算
- numpy softmax 广播 + Python GIL

**优化空间**：
- Cython/pybind11：**-60%**（→ 110 ms）
- 挪进 RKNN 模型：**-90%**（→ 30 ms）
- NumPy 向量化优化：**-20%**

### Stage 5 · Rotated NMS（1.8 ms）

**近似中心距离 NMS**（见 `common.py::nms_rot`）：

```python
# 按 conf 降序，保留 top；
# 剩余中心距离 > (w+h)*0.5*0.3 的才保留
```

O(N²) 但 N=407，很快。真正的 `cv2.rotatedRectangleIntersection` 慢 20×。
