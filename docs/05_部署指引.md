# 05 · 部署指引

## 🚀 快速开始

### 前置条件

- **硬件**：Orange Pi 5 / 5 Plus / 任何 RK3588 单板
- **系统**：Ubuntu 22.04 (armbian / orangepi os)
- **Python**：3.8+ （建议 3.10）
- **RKNN 运行时**：rknn-toolkit-lite2 v1.5.2+

### 安装依赖

```bash
# 1. RKNN Lite 运行时
pip install rknn-toolkit-lite2

# 2. OpenCV + numpy
pip install opencv-python numpy

# 3. （板子上必须）驱动已预装
# /usr/lib/librknnrt.so 应该存在
ls /usr/lib/librknnrt.so
```

如果没有 librknnrt.so：

```bash
# 下载 Rockchip 官方 runtime
wget https://github.com/airockchip/rknn-toolkit2/raw/master/runtime/Linux/librknn_api/aarch64/librknnrt.so
sudo cp librknnrt.so /usr/lib/
```

## 🔌 集成到业务代码

### 最小示例

```python
from production_v2 import ProductionPipeline
import cv2

pipe = ProductionPipeline()   # 初始化 3 核 + warmup（只做一次）

# 主循环
while True:
    img = cv2.imread("some_8k.jpg")   # 或 frame = cap.read()
    final_dets, stats = pipe.run(img, conf=0.15)
    
    # final_dets: List[[cx, cy, w, h, angle, conf, cls]]
    # stats['wall_all_parallel']: ~800ms
    
    for d in final_dets:
        cx, cy, w, h, angle, conf, cls = d
        # 你的业务逻辑（画框、计数、追踪...）
    
pipe.close()
```

### 参数调节

| 参数 | 默认 | 可选范围 | 说明 |
|---|---|---|---|
| `TILE_CUT` | 2048 | 1024/2048/4096 | 切片大小，建议不动 |
| `TILE_MODEL` | 1024 | 640/1024 | 模型原生尺寸 |
| `OVERLAP` | 200 | 100-500 | 切片重叠 |
| `CONF` | 0.15 | 0.1-0.5 | 可调（越低召回越高）|
| `RESIZE_ALGO` | LINEAR | LINEAR/CUBIC | CUBIC 精度稍高但 +80ms |
| `N_CORES` | 3 | 1/2/3 | 建议 3 |

## 🛠️ 如果要处理不同尺寸的图

### 处理 6K × 6K 图

```python
# 切片规则会自动适配，不需要改代码
img = cv2.imread("6k.jpg")  # shape (6144, 6144, 3)
final, stats = pipe.run(img)
# tile 数从 25 降到 ~16，时间约 550ms
```

### 处理 10K × 10K 以上的图

```python
# 切片数会增多，时间线性扩张
# 10K → 36 tile ~ 1200ms（超 1s，需要优化）

# 解决方案 A：先 resize 到 8192 再推理（会损失小目标）
img_small = cv2.resize(img, (8192, 8192))

# 解决方案 B：两遍推理（上半图 + 下半图）
top = img[:8192, :]
bot = img[4096:, :]
final_top = pipe.run(top)
final_bot = pipe.run(bot)
# 合并 + 调整 y 坐标
```

### 处理非正方形图（比如 8192 × 5000）

```python
# 直接塞进 pipe.run(img) 就行
# gen_tiles 自动按 W / H 分别规划
```

## 🧪 验证部署正确

### 测试 1：模型 cls 通道没塌

```python
from rknnlite.api import RKNNLite
import numpy as np

r = RKNNLite()
r.load_rknn("models/yolov8n-obb_i8_1024_airockchip.rknn")
r.init_runtime(core_mask=RKNNLite.NPU_CORE_0)

# 用真实图
img = cv2.imread("test_8k.jpg")
crop = cv2.resize(img[0:2048, 0:2048], (1024, 1024))
rgb = cv2.cvtColor(crop, cv2.COLOR_BGR2RGB)
outs = r.inference(inputs=[np.expand_dims(rgb, 0)], data_format="nhwc")

cls_logits = outs[0][0, 64:79]   # (15, 128, 128)
print(f"cls min: {cls_logits.min():.2f}, max: {cls_logits.max():.2f}")
# 期望：min 约 -5~-10，max 约 5~10（非全 0）
```

### 测试 2：单次推理时间

```python
import time
from code.production_v2 import ProductionPipeline

pipe = ProductionPipeline()
img = cv2.imread("test_8k.jpg")

# warmup
for _ in range(3):
    pipe.run(img, conf=0.15)

# benchmark
times = []
for _ in range(10):
    t0 = time.perf_counter()
    pipe.run(img, conf=0.15)
    times.append((time.perf_counter()-t0)*1000)

import numpy as np
print(f"median: {np.median(times):.0f} ms")  # 期望 ~815ms
```

### 测试 3：Recall 一致

运行 `code/production_v2.py`，期望 recall 稳定 63.8%（±0%）。

如果 recall 波动很大：
- 检查 GT 是不是用同一张图生成
- 检查 random seed（本方案全部确定性，不该有随机）

## 🔧 常见问题

### Q1: 跑不起来 RKNN runtime

```
E init_runtime: load rknpu runtime failed!
```

**解**：

```bash
# 检查 librknnrt.so
ls -la /usr/lib/librknnrt.so
# 检查 NPU 驱动版本
cat /sys/kernel/debug/rknpu/version   # 期望 >= 0.9.6
```

### Q2: cls 全 0

看 `02_模型来历_量化步骤.md` 的"陷阱"章节。必须用 airockchip 分支导出的 ONNX。

### Q3: 跑得比 815ms 慢

可能原因：

1. **核数没满**：检查 `build_ctx_list(MODEL_1024, 3)` 传的是 3
2. **CPU 被占用**：运行前 `htop` 看没有其他高 CPU 进程
3. **NPU 频率降了**：
   ```bash
   sudo cat /sys/class/devfreq/fdab0000.npu/cur_freq
   # 期望 1000000000 (1GHz)
   # 如果低，强制最高：
   echo performance | sudo tee /sys/class/devfreq/fdab0000.npu/governor
   ```
4. **温度降频**：`cat /sys/class/thermal/thermal_zone*/temp`，超过 85°C 会降频

### Q4: 如何看自己的 NPU 性能是不是正常

```bash
python -c "
from rknnlite.api import RKNNLite
import numpy as np, time
r = RKNNLite()
r.load_rknn('models/yolov8n-obb_i8_1024_airockchip.rknn')
r.init_runtime(core_mask=RKNNLite.NPU_CORE_0)
dummy = np.zeros((1,1024,1024,3), dtype=np.uint8)
for _ in range(3): r.inference(inputs=[dummy], data_format='nhwc')
ts = []
for _ in range(10):
    t=time.perf_counter(); r.inference(inputs=[dummy], data_format='nhwc'); ts.append((time.perf_counter()-t)*1000)
print(f'single-core inf: median {np.median(ts):.1f} ms (expect ~55-65)')
"
```

### Q5: 召回低于 63.8%

可能：

1. **测试图不是 test_8k.jpg**：不同图 GT 不同
2. **conf 调高了**：确保 0.15
3. **NMS IoU 改过**：默认 0.3，不要改
