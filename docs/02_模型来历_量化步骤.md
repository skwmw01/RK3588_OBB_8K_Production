# 02 · 模型来历 & 量化步骤

## 🌳 模型血统

### 基座：YOLOv8n-OBB (Ultralytics)

- **来源**: Ultralytics 官方 https://github.com/ultralytics/ultralytics
- **任务**: OBB = Oriented Bounding Box，旋转框检测
- **主干**: YOLOv8 标准 CSPDarknet + PAN-FPN
- **Head**: OBB 专用解耦头（分类 + 回归 + 角度）
- **规模**: n 级（nano），约 3.0 M 参数

### 训练数据集：DOTA v1.0

- **DOTA** = Dataset of Object deTection in Aerial images，航拍旋转框标杆数据集
- 官方站: https://captain-whu.github.io/DOTA/
- 15 个类别：飞机 / 船 / 油罐 / 桥 / 车辆 / 运动场 / 机场 / 港口 / …
- 图像尺寸：800×800 到 20000×20000 不等
- 标注：4 点多边形 → `(cx, cy, w, h, angle)` 旋转框
- Ultralytics 发布的 `yolov8n-obb.pt` 就是在 DOTA v1.0 上训的

### 为什么用 "n" 级最小模型？

| 模型 | 参数量 | @1024 单核速度 | RKNN INT8 大小 |
|---|---|---|---|
| yolov8n-obb | 3.0 M | **65 ms** | **4.6 MB** ✅ 选它 |
| yolov8s-obb | 11.4 M | ~220 ms | ~15 MB ❌ 超 1s |
| yolov8m-obb | 26.5 M | ~520 ms | ~32 MB ❌ 完全不行 |

**在 RK3588 上跑 25 tile 推理，只有 n 级能在 1s 内搞定。**

## 🕳️ 陷阱：官方导出的 ONNX 量化会塌方

### 症状

用 Ultralytics 官方 `model.export(format='onnx', imgsz=1024)` 导出的 ONNX：

- Head 会把 3 个 stride 的输出 concat 成 **`(1, 20, 21504)`** 单输出
- 20 = 4(box) + 15(cls) + 1(angle)，21504 = anchor 总数

**问题**：这个 concat 后的巨大 tensor 被 RKNN INT8 量化：

- cls 通道会整体缩放抹平（**归零**）
- 推理输出全是 `conf ≈ 0`，**完全没有检测框**

**原因**：
- Box 通道数值范围 0-10（DFL bins）
- cls 通道是 logits，-20 ~ +20
- concat 到同一个 tensor 后共享 scale → cls 动态范围被 box 压死

### 验证踩坑（2026-04-22 当天实测）

尝试过 4 种量化配置，全部塌：

- `quantized_algorithm='normal'` → cls 全 0
- `quantized_algorithm='mmse'` → cls 全 0
- `quantized_algorithm='kl_divergence'` + per-channel → cls 全 0
- 多样性 calibration（50 张 DOTA） → cls 全 0

**结论：官方 ONNX 在 RK3588 INT8 下 100% 塌方。**

## ✅ 解决方案：airockchip 分支

### 来源

- GitHub: https://github.com/airockchip/ultralytics_yolov8
- 瑞芯微官方 fork 的 Ultralytics 分支
- 专门为 RKNN 导出优化过 YOLOv8 的 head
- 版本锁定：
  - yolov8 → commit `v8.2.82`
  - yolo11 → commit `v8.3.7`
  - 版本不对转 ONNX 会报错

### 关键魔改

在 `ultralytics/nn/modules/head.py` 里改 OBB head：

```python
# 官方：3 个 stride 输出 concat 成一个巨大 tensor
# ❌ return torch.cat([out0, out1, out2], dim=-1)   # (1, 20, 21504)

# ✅ airockchip format='rknn'：分离成 4 个输出
return [
    out0,      # (1, 79, 128, 128)  stride=8
    out1,      # (1, 79, 64, 64)    stride=16
    out2,      # (1, 79, 32, 32)    stride=32
    angle,     # (1, 1, 21504)      pre-sigmoid
]
# 79 = 64 (4 box × 16 DFL bins) + 15 cls logits
```

**为什么这样能救 INT8**：

- 3 个 stride 的 box / cls 在量化时独立 scale
- cls 通道不会被 box 压死
- DFL 16 bins 保留在模型里

**注意**：

- angle 在 format='rknn' 已经做了 sigmoid（值域 0-1）
- cls 是 raw logits，**host 端 decode 时要自己 sigmoid**
- 见 `code/common.py` 的 `decode_generic`

## 🔧 完整量化流程（pt → rknn）

### Step 1: 拉 airockchip + 安装

```bash
git clone https://github.com/airockchip/ultralytics_yolov8.git
cd ultralytics_yolov8
git checkout v8.2.82
pip install -e .
```

### Step 2: 导出 ONNX

编辑 `ultralytics/cfg/default.yaml`：

```yaml
task: obb
mode: export
model: yolov8n-obb.pt
imgsz: 1024
format: rknn      # 关键！不要用 format=onnx
```

然后：

```bash
PYTHONPATH=./ python ./ultralytics/engine/exporter.py
# 输出：yolov8n-obb.onnx (4 输出分离头版)
```

### Step 3: 准备 calibration 数据

```bash
mkdir calib
# 从 DOTA 数据集拷 10 张真实图到 calib/
ls -d $(pwd)/calib/*.jpg > dataset.txt
```

Calibration 数量建议：

- 最少 5 张
- 推荐 10-20 张
- 太多反而让 INT8 过于"泛化"

### Step 4: RKNN 量化

见 `quantize/convert_airockchip_v8.py`：

```python
from rknn.api import RKNN

r = RKNN(verbose=False)
r.config(
    mean_values=[[0, 0, 0]],
    std_values=[[255, 255, 255]],
    target_platform="rk3588",
    quantized_algorithm="normal",
    optimization_level=3,
)
r.load_onnx(model="yolov8n-obb.onnx")
r.build(do_quantization=True, dataset="dataset.txt")
r.export_rknn("yolov8n-obb_i8_1024_airockchip.rknn")
r.release()
```

**关键参数**：

| 参数 | 值 | 说明 |
|---|---|---|
| mean_values | [0,0,0] | 不做减均值 |
| std_values | [255,255,255] | 输入 / 255 归一化到 [0,1] |
| target_platform | "rk3588" | 3 核 NPU |
| quantized_algorithm | "normal" | 标准 per-tensor INT8 (mmse/kl 会塌) |
| optimization_level | 3 | 最高，启 ops fusion |

运行时间：2-5 分钟。

### Step 5: 验证没塌

```python
from rknnlite.api import RKNNLite
r = RKNNLite()
r.load_rknn("yolov8n-obb_i8_1024_airockchip.rknn")
r.init_runtime(core_mask=RKNNLite.NPU_CORE_0)
outs = r.inference(inputs=[test_image], data_format="nhwc")
cls_logits = outs[0][0, 64:79]  # (15, H, W)
print("cls min:", cls_logits.min(), "max:", cls_logits.max())
# 期望 min ≈ -5~-10, max ≈ 5~10（非全 0）
```

**如果 cls 全 0**：

1. 检查用了 airockchip 分支（format=rknn）
2. 检查 ONNX 有 4 个输出（不是 1 个 concat）
3. 换 calibration 图片再试
